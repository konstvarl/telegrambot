# Hotel Search Bot for Telegram

[![Python](https://img.shields.io/badge/Python-3.10-blue.svg)](https://www.python.org/downloads/release/python-3100/) [![pyTelegramBotAPI](https://img.shields.io/badge/pyTelegramBotAPI-4.4.0-blue.svg)](https://pypi.org/project/pyTelegramBotAPI/) [![Peewee](https://img.shields.io/badge/Peewee-3.14.4-blue.svg)](https://pypi.org/project/peewee/) [![License](https://img.shields.io/badge/License-MIT-green.svg)](https://opensource.org/licenses/MIT)

> **⚠️ Важное примечание**
>
> Этот бот использует **тестовую среду** Amadeus for Developers self-service APIs. Это означает, что все данные об отелях, ценах и доступности могут быть неактуальными и не полными. Проект предназначен исключительно для демонстрации технических возможностей и не должен использоваться для реального планирования поездок.

**Hotel Search Bot** — это Telegram-бот ищет отели по всему миру. Бот ведёт пошаговый диалог, в ходе которого пользователь указывает все необходимые критерии для поиска.

Этот проект демонстрирует техники разработки Telegram-ботов, включая управление состояниями (FSM), фоновую обработку задач, кэширование, парсинг веб-страниц и построение отказоустойчивой архитектуры.

## 🚀 Основные возможности

*   **Три сценария поиска**:
    *   `/lowprice` — самые дешевые отели.
    *   `/guest_rating` — самые популярные отели.
    *   `/bestdeal` — расположенные ближе к центру города.
*   **Гибкие критерии поиска**: Пользователь может указать город, даты заезда и выезда (с помощью интерактивного календаря), диапазон цен и расстояние до центра.
*   **Отображение фотографий**: Показывается до 50 фотографий отеля.
*   **История запросов**: Команда `/history` позволяет просмотреть последние успешные поисковые запросы.
*   **Пагинация**: Переключение между найденными отелями и их фотографиями с помощью инлайн-кнопок.
*   **Локализация**: Весь интерфейс, включая календарь, представлен на русском языке.

## 🛠️ Технологический стек

*   **Язык**: Python 3.10
*   **Telegram API**: `pyTelegramBotAPI`
*   **База данных**: `SQLite` (через ORM `Peewee`)
*   **Внешние API**:
    *   `Amadeus for Developers self-service APIs` — для поиска отелей и предложений.
*   **Ключевые библиотеки**:
    *   `python-dotenv` — для управления конфигурацией.
    *   `telegram-bot-calendar` — для интерактивного календаря.
    *   `pycountry`, `countryinfo` — для получения дополнительной информации о странах.

## ✨ Ключевые технические решения

Этот проект — не просто обертка над API. В нем реализованы следующие механики:

1.  **Архитектура на базе FSM**
    *   Логика бота построена на машине состояний (`StatesGroup`), что позволяет вести пользователя по сложному сценарию опроса, сохраняя контекст между шагами.
    *   Код четко разделен на слои: обработчики (`handlers`), работа с БД (`database`), вызовы API (`api`), утилиты (`utils`).

2.  **Асинхронная фоновая загрузка фотографий**
    *   Поиск и загрузка фотографий отелей выполняются в отдельных потоках (`threading`), не блокируя основной процесс бота.
    *   Реализован механизм отмены фоновых задач: при переключении на следующий отель загрузка фото для текущего прерывается.

3.  **Система кэширования**
    *   Все запросы к внешним API (Amadeus, поиск фото) кэшируются в базе данных с помощью декоратора `@api_cache`.
    *   Кэш имеет время жизни (TTL) и использует вероятностную очистку для удаления устаревших записей без ущерба для производительности.

4.  **Кастомный веб-парсер для поиска фото**
    *   Поскольку Amadeus API не предоставляет фото отелей, был написан собственный парсер поисковой выдачи DuckDuckGo.
    *   Парсер эмулирует запросы внутреннего API DuckDuckGo и использует систему оценки релевантности, чтобы отфильтровать "мусорные" картинки и найти наиболее подходящие фотографии отеля.

5.  **Отказоустойчивость при работе с Telegram API**
    *   Написан набор "безопасных" функций-оберток (`safe_edit_message`, `safe_delete_message`), которые перехватывают частые исключения Telegram API (например, `message is not modified` или попытка отредактировать удаленное сообщение) и предотвращают падение бота.

6.  **Защита от "протухших" инлайн-кнопок**
    *   Реализован механизм сессий. Каждому поиску присваивается уникальный ID, который "зашивается" в `callback_data` инлайн-кнопок. Специальный декоратор `@require_valid_session` не позволяет пользователю взаимодействовать с кнопками от старых, уже неактуальных поисковых запросов.

## ⚙️ Установка и запуск

1.  **Клонируйте репозиторий:**
    ```sh
    git clone https://github.com/konstvarl/hotel-search-bot.git
    cd hotel-search-bot
    ```

2.  **Создайте и активируйте виртуальное окружение:**
    ```sh
    python -m venv venv
    source venv/bin/activate  # Для Windows: venv\Scripts\activate
    ```

3.  **Установите зависимости:**
    ```sh
    pip install -r requirements.txt
    ```

4.  **Настройте переменные окружения:**
    *   Создайте файл `.env` в корневой папке проекта, скопировав из `.env.template`.
    *   Заполните его вашими данными:
    ```
    # .env

    # Токен вашего Telegram-бота (получить у @BotFather)
    BOT_TOKEN="12345:ABC-DEF1234ghIkl-zyx57W2v1u123ew11"

    # Ключ и секрет для Amadeus for Developers API
    AMADEUS_API_KEY="YourAmadeusApiKey"
    AMADEUS_API_SECRET="YourAmadeusApiSecret"
    ```

5.  **Запустите бота:**
    ```sh
    python main.py
    ```
